# Domains

GuildServer allows you to attach **custom domains** to your applications. Each domain is verified, secured with automatic SSL certificates from Let's Encrypt, and routed through Traefik to the correct container.

## How Domains Work

When you add a domain to an application, GuildServer:

1. Generates a **verification token** for DNS validation.
2. Waits for you to add a DNS record (CNAME or TXT) pointing to GuildServer.
3. Verifies the DNS record matches the expected token.
4. Provisions an **SSL certificate** via Let's Encrypt (for non-localhost domains).
5. Configures **Traefik routing** to forward traffic for that domain to the application container.

## Domain Properties

| Field | Description |
|-------|-------------|
| `domain` | The fully qualified domain name (e.g., `app.example.com`) |
| `applicationId` | The application this domain routes to |
| `isPrimary` | Whether this is the primary domain for the application |
| `isAutoGenerated` | Whether this domain was auto-generated by the platform |
| `verificationToken` | Token for DNS verification |
| `verificationMethod` | DNS verification method: `cname` or `txt` |
| `verified` | Whether DNS verification has passed |
| `status` | `pending`, `verifying`, `active`, `failed`, `expired` |
| `forceHttps` | Whether to redirect HTTP to HTTPS (default: `true`) |

## Auto-Generated Domains

When an application is deployed, GuildServer automatically generates a subdomain based on the `appName` and the platform's `BASE_DOMAIN`:

```
<app-name>.<base-domain>
```

For example, with `BASE_DOMAIN=guildserver.localhost`, an application named `my-api` gets `my-api.guildserver.localhost`.

## DNS Verification

For custom domains, you need to verify ownership by adding a DNS record:

**CNAME Verification (Recommended):**
```
app.example.com  CNAME  guildserver.yourserver.com
```

**TXT Verification:**
```
_guildserver-verify.app.example.com  TXT  "gs-verify=<token>"
```

## SSL Certificates

GuildServer uses **Let's Encrypt** for automatic SSL certificate provisioning via Traefik's ACME integration. Certificates are automatically provisioned when a domain is verified, automatically renewed before expiration, and stored in the Traefik ACME storage.
